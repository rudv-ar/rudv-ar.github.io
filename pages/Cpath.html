Add this file to your LazyVim extras:

{ import = "lazyvim.plugins.extras.lang.clangd" }

Debugging

GDB essentials + memory checking with Valgrind.

Compile with Debug Symbols

Always use -g (and disable optimizations with -O0) when debugging.

bash 
$ gcc -g -O0 -Wall -Wextra -o myprog myprog.c

GDB Quick Session

gdb session 
$ gdb ./myprog
(gdb) break main         # breakpoint at main
(gdb) run                # start program
(gdb) next               # step over (n)
(gdb) step               # step into (s)
(gdb) print my_var       # inspect variable
(gdb) backtrace          # show call stack (bt)
(gdb) continue           # resume (c)
(gdb) quit               # exit

Valgrind — Memory Leak Detection

bash 
$ valgrind --leak-check=full --show-leak-kinds=all ./myprog
==1234== HEAP SUMMARY:
==1234==   in use at exit: 0 bytes in 0 blocks
==1234==   total heap usage: 3 allocs, 3 frees
==1234== All heap blocks were freed -- no leaks are possible ✓

AddressSanitizer (faster than Valgrind)

Compile-time instrumentation for catching memory bugs at runtime — no separate tool needed.

bash 
$ gcc -g -fsanitize=address,undefined -o myprog myprog.c
$ ./myprog   # ASAN output on violations
Build Systems

From a single Makefile to full CMake projects.

The Essential Makefile

For small-to-medium projects, a well-structured Makefile is all you need.

Makefile 
CC      = gcc
CFLAGS  = -Wall -Wextra -Wpedantic -std=c11 -O2
DBGFLAGS= -g -O0 -fsanitize=address,undefined
SRC     = $(wildcard src/*.c)
OBJ     = $(SRC:.c=.o)
TARGET  = myprog

all: $(TARGET)

$(TARGET): $(OBJ)
    $(CC) $(CFLAGS) -o $@ $^

debug: CFLAGS = $(DBGFLAGS)
debug: $(TARGET)

clean:
    rm -f $(OBJ) $(TARGET)

.PHONY: all debug clean

CMake Project Structure

For larger projects with dependencies, CMake + Ninja is the modern standard.

CMakeLists.txt 
cmake_minimum_required(VERSION 3.20)
project(MyProject C)
set(CMAKE_C_STANDARD 11)
set(CMAKE_EXPORT_COMPILE_COMMANDS ON)  # for clangd!

add_executable(myprog src/main.c src/utils.c)
target_compile_options(myprog PRIVATE
    -Wall -Wextra -Wpedantic)
bash · build commands 
$ cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Debug
$ cmake --build build       # or: ninja -C build
$ # symlink compile_commands for clangd:
$ ln -s build/compile_commands.json .

Recommended Compiler Flags

Handy .bashrc Aliases

~/.bashrc 
# Quick compile alias
alias cc='gcc -Wall -Wextra -std=c11'
alias ccd='gcc -g -O0 -fsanitize=address,undefined'

# One-liner: compile & run
function crun() {
  gcc -Wall -Wextra -std=c11 -o /tmp/crun "$1" && /tmp/crun
}

weeks 5–8

Pro tip: Use man 3 <function> for C library functions and man 2 <syscall> for Linux syscalls. Install man-pages if needed: sudo pacman -S man-pages
